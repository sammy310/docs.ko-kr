# <a name="classification-of-standard-query-operators-by-manner-of-execution-c"></a><span data-ttu-id="85eaf-102">실행 방식에 따라 표준 쿼리 연산자 분류(C#)</span><span class="sxs-lookup"><span data-stu-id="85eaf-102">Classification of Standard Query Operators by Manner of Execution (C#)</span></span>
<span data-ttu-id="85eaf-103">표준 쿼리 연산자 메서드의 LINQ to Objects 구현은 즉시 실행 또는 지연된 실행의 두 가지 기본 방식 중 하나로 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="85eaf-104">지연된 실행을 사용하는 쿼리 연산자는 스트리밍 및 비스트리밍의 두 가지 범주로 추가로 구분할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="85eaf-105">여러 쿼리 연산자가 어떻게 실행되는지 알고 있으면 제공된 쿼리에서 얻을 결과를 이해하는 데 도움이 될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="85eaf-106">데이터 소스가 변경되거나 다른 쿼리 위에 쿼리를 빌드할 경우 특히 도움이 됩니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="85eaf-107">이 항목에서는 실행 방식에 따라 표준 쿼리 연산자를 분류합니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="85eaf-108">실행 방식</span><span class="sxs-lookup"><span data-stu-id="85eaf-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="85eaf-109">직접 실행</span><span class="sxs-lookup"><span data-stu-id="85eaf-109">Immediate</span></span>  
 <span data-ttu-id="85eaf-110">즉시 실행은 코드의 쿼리가 선언되는 지점에서 데이터 소스를 읽고 작업이 수행됨을 의미합니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="85eaf-111">하나의 비열거형 결과를 반환하는 모든 표준 쿼리 연산자는 즉시 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="85eaf-112">연기됨</span><span class="sxs-lookup"><span data-stu-id="85eaf-112">Deferred</span></span>  
 <span data-ttu-id="85eaf-113">지연된 실행은 코드의 쿼리가 선언되는 지점에서 작업이 수행되지 않음을 의미합니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="85eaf-114">예를 들어 `foreach` 문을 사용하여 쿼리 변수가 열거될 경우에만 작업이 수행됩니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="85eaf-115">즉, 쿼리 실행 결과는 쿼리가 정의될 때가 아니라 쿼리가 실행될 때 데이터 소스의 내용에 따라 달라집니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="85eaf-116">쿼리 변수가 여러 번 열거될 경우 매번 결과가 다를 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="85eaf-117">반환 형식이 <xref:System.Collections.Generic.IEnumerable%601> 또는 <xref:System.Linq.IOrderedEnumerable%601>인 표준 쿼리 연산자는 대부분 지연 방식으로 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="85eaf-118">지연된 실행을 사용하는 쿼리 연산자는 스트리밍 및 비스트리밍으로 추가로 분류할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="85eaf-119">스트리밍</span><span class="sxs-lookup"><span data-stu-id="85eaf-119">Streaming</span></span>  
 <span data-ttu-id="85eaf-120">스트리밍 연산자는 요소를 생성하기 전에 모든 소스 데이터를 읽을 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="85eaf-121">실행 시 스트리밍 연산자는 소스 요소를 읽을 때 각 소스 요소에 대해 작업을 수행하고 해당하는 경우 요소를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="85eaf-122">스트리밍 연산자는 결과 요소가 생성될 때까지 소스 요소를 계속 읽습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="85eaf-123">즉, 두 개 이상의 소스 요소를 읽어 하나의 결과 요소를 생성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="85eaf-124">비스트리밍</span><span class="sxs-lookup"><span data-stu-id="85eaf-124">Non-Streaming</span></span>  
 <span data-ttu-id="85eaf-125">비스트리밍 연산자는 결과 요소를 생성하기 전에 모든 소스 데이터를 읽어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="85eaf-126">정렬 또는 그룹화 등의 작업은 이 범주로 분류됩니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="85eaf-127">실행 시 비스트리밍 쿼리 연산자는 모든 소스 데이터를 읽고, 데이터 구조에 넣고, 작업을 수행하고, 결과 요소를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="85eaf-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
