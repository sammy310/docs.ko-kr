<span data-ttu-id="106e8-153">`My` 네임스페이스의 확장을 개발하는 경우 다음 지침을 사용하여 확장 구성 요소의 유지 관리 비용을 최소화합니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-153">When you develop extensions to the `My` namespace, use the following guidelines to help minimize the maintenance costs of your extension components:</span></span>

## <a name="designing-class-libraries-for-my"></a><span data-ttu-id="106e8-165">`My`용 클래스 라이브러리 디자인</span><span class="sxs-lookup"><span data-stu-id="106e8-165">Designing class libraries for `My`</span></span>

<span data-ttu-id="106e8-166">대부분의 개체 모델과 마찬가지로 일부 디자인 패턴은 `My` 네임스페이스에서 잘 작동하고 다른 디자인 패턴은 그렇지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-166">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="106e8-167">`My` 네임스페이스에 대한 확장을 디자인하는 경우 다음 원칙을 고려하세요.</span><span class="sxs-lookup"><span data-stu-id="106e8-167">When designing an extension to the `My` namespace, consider the following principles:</span></span>

- <span data-ttu-id="106e8-168">**상태 비저장 메서드.**</span><span class="sxs-lookup"><span data-stu-id="106e8-168">**Stateless methods.**</span></span> <span data-ttu-id="106e8-169">`My` 네임스페이스의 메서드는 특정 작업에 대한 완전한 솔루션을 제공해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-169">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="106e8-170">메서드에 전달된 매개 변수 값이 특정 작업을 완료하는 데 필요한 모든 입력을 제공하는지 확인합니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-170">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="106e8-171">리소스에 대한 열린 연결 등 이전 상태를 사용하는 메서드를 만들지 않아야 합니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-171">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>

<span data-ttu-id="106e8-184">다음 지침에서는 클래스 라이브러리에 대한 일반적인 디자인 원칙을 배제하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-184">These guidelines do not preclude general design principles for class libraries.</span></span> <span data-ttu-id="106e8-185">오히려 Visual Basic 및 `My` 네임스페이스를 사용하는 개발자에게 최적화된 권장 사항입니다.</span><span class="sxs-lookup"><span data-stu-id="106e8-185">Rather, they are recommendations that are optimized for developers who are using Visual Basic and the `My` namespace.</span></span> <span data-ttu-id="106e8-186">클래스 라이브러리 만들기에 대한 일반적인 디자인 원칙은 [프레임워크 디자인 지침](../../../standard/design-guidelines/index.md)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="106e8-186">For general design principles for creating class libraries, see [Framework Design Guidelines](../../../standard/design-guidelines/index.md).</span></span>
