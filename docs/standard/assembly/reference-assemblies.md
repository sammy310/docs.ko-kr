# <a name="reference-assemblies"></a><span data-ttu-id="22347-103">참조 어셈블리</span><span class="sxs-lookup"><span data-stu-id="22347-103">Reference assemblies</span></span>

<span data-ttu-id="22347-104">*참조 어셈블리*는 라이브러리의 공용 API 화면을 나타내는 데 필요한 최소한의 메타데이터만 포함하는 특수한 형식의 어셈블리입니다.</span><span class="sxs-lookup"><span data-stu-id="22347-104">*Reference assemblies* are a special type of assembly that contain only the minimum amount of metadata required to represent the library's public API surface.</span></span> <span data-ttu-id="22347-105">빌드 도구에서 어셈블리를 참조할 때 중요한 모든 멤버에 대한 선언을 포함하지만, 해당 API 계약에 영향을 미치지 않는 프라이빗 멤버의 선언과 모든 멤버 구현은 제외됩니다.</span><span class="sxs-lookup"><span data-stu-id="22347-105">They include declarations for all members that are significant when referencing an assembly in build tools, but exclude all member implementations and declarations of private members that have no observable impact on their API contract.</span></span> <span data-ttu-id="22347-106">반면 일반 어셈블리는 *구현 어셈블리*라고 합니다.</span><span class="sxs-lookup"><span data-stu-id="22347-106">In contrast, regular assemblies are called *implementation assemblies*.</span></span>

<span data-ttu-id="22347-107">참조 어셈블리는 실행을 위해 로드할 수 없지만 구현 어셈블리와 동일한 방식으로 컴파일러 입력으로 전달될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-107">Reference assemblies can't be loaded for execution, but they can be passed as compiler input in the same way as implementation assemblies.</span></span> <span data-ttu-id="22347-108">참조 어셈블리는 일반적으로 특정 플랫폼 또는 라이브러리의 SDK(소프트웨어 개발 키트)를 사용하여 배포됩니다.</span><span class="sxs-lookup"><span data-stu-id="22347-108">Reference assemblies are usually distributed with the Software Development Kit (SDK) of a particular platform or library.</span></span>

<span data-ttu-id="22347-109">참조 어셈블리를 사용하면 개발자는 해당 버전에 대한 전체 구현 어셈블리 없이도 특정 라이브러리 버전을 대상으로 하는 프로그램을 빌드할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-109">Using a reference assembly enables developers to build programs that target a specific library version without having the full implementation assembly for that version.</span></span> <span data-ttu-id="22347-110">컴퓨터에 최신 버전의 라이브러리만 있지만 해당 라이브러리의 이전 버전을 대상으로 하는 프로그램을 빌드하려고 한다고 가정해 봅니다.</span><span class="sxs-lookup"><span data-stu-id="22347-110">Suppose, you have only the latest version of some library on your machine, but you want to build a program that targets an earlier version of that library.</span></span> <span data-ttu-id="22347-111">구현 어셈블리에 대해 직접 컴파일하는 경우 이전 버전에서 사용할 수 없는 API 멤버를 예기치 않게 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-111">If you compile directly against the implementation assembly, you might inadvertently use API members that aren't available in the earlier version.</span></span> <span data-ttu-id="22347-112">대상 컴퓨터에서 프로그램을 테스트하는 경우에만 이 오류를 찾을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-112">You'll only find this mistake when testing the program on the target machine.</span></span> <span data-ttu-id="22347-113">이전 버전의 참조 어셈블리에 대해 컴파일하는 경우 컴파일 시간 오류가 즉시 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="22347-113">If you compile against the reference assembly for the earlier version, you'll immediately get a compile-time error.</span></span>

<span data-ttu-id="22347-114">또한, 참조 어셈블리는 계약, 즉 구체적인 구현 어셈블리에 해당하지 않는 API 집합을 나타낼 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-114">A reference assembly can also represent a contract, that is, a set of APIs that don't correspond to the concrete implementation assembly.</span></span> <span data-ttu-id="22347-115">*계약 어셈블리*라고 하는 이러한 참조 어셈블리는 동일한 API 집합을 지원하는 여러 플랫폼을 대상으로 지정하는 데 사용될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-115">Such reference assemblies, called the *contract assembly*, can be used to target multiple platforms that support the same set of APIs.</span></span> <span data-ttu-id="22347-116">예를 들어 .NET Standard는 서로 다른 .NET 플랫폼 간에 공유되는 공통 API 집합을 나타내는 계약 어셈블리 *netstandard.dll*을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="22347-116">For example, .NET Standard provides the contract assembly, *netstandard.dll*, that represents the set of common APIs shared between different .NET platforms.</span></span> <span data-ttu-id="22347-117">이러한 API의 구현은 .NET Framework의 *mscorlib.dll* 또는 .NET Core의 *System.Private.CoreLib.dll*과 같이 다른 플랫폼의 다른 어셈블리에 포함되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-117">The implementations of these APIs are contained in different assemblies on different platforms, such as *mscorlib.dll* on .NET Framework or *System.Private.CoreLib.dll* on .NET Core.</span></span> <span data-ttu-id="22347-118">.NET Standard를 대상으로 하는 라이브러리는 .NET Standard를 지원하는 모든 플랫폼에서 실행될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-118">A library that targets .NET Standard can run on all platforms that support .NET Standard.</span></span>

## <a name="reference-assemblies-structure"></a><span data-ttu-id="22347-154">참조 어셈블리 구조</span><span class="sxs-lookup"><span data-stu-id="22347-154">Reference assemblies structure</span></span>

<span data-ttu-id="22347-155">참조 어셈블리는 관련 개념인 *메타데이터 전용 어셈블리*의 확장입니다.</span><span class="sxs-lookup"><span data-stu-id="22347-155">Reference assemblies are an expansion of the related concept, *metadata-only assemblies*.</span></span> <span data-ttu-id="22347-156">메타데이터 전용 어셈블리에는 단일 `throw null` 본문으로 대체되는 메서드 본문이 있지만 익명 형식을 제외한 모든 멤버가 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="22347-156">Metadata-only assemblies have their method bodies replaced with a single `throw null` body, but include all members except anonymous types.</span></span> <span data-ttu-id="22347-157">본문이 없는 경우와 대조적으로 `throw null` 본문을 사용하는 이유는 **PEVerify**가 실행 및 전달될 수 있도록 하여 메타데이터의 완전성을 검증하기 위한 것입니다.</span><span class="sxs-lookup"><span data-stu-id="22347-157">The reason for using `throw null` bodies (as opposed to no bodies) is so that **PEVerify** can run and pass (thus validating the completeness of the metadata).</span></span>

<span data-ttu-id="22347-158">참조 어셈블리는 메타데이터 프라이빗 어셈블리에서 메타데이터(전용 멤버)를 추가로 제거합니다.</span><span class="sxs-lookup"><span data-stu-id="22347-158">Reference assemblies further remove metadata (private members) from metadata-only assemblies:</span></span>

- <span data-ttu-id="22347-159">참조 어셈블리에는 API 화면에 있어야 하는 항목에 대한 참조만 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="22347-159">A reference assembly only has references for what it needs in the API surface.</span></span> <span data-ttu-id="22347-160">실제 어셈블리에는 특정 구현에 관련된 추가 참조가 포함될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-160">The real assembly may have additional references related to specific implementations.</span></span> <span data-ttu-id="22347-161">예를 들어, `class C { private void M() { dynamic d = 1; ... } }`에 대한 참조 어셈블리는 `dynamic`에 필요한 형식을 참조하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-161">For instance, the reference assembly for `class C { private void M() { dynamic d = 1; ... } }` doesn't reference any types required for `dynamic`.</span></span>

<span data-ttu-id="22347-164">참조 어셈블리의 메타데이터는 다음 정보를 계속 유지합니다.</span><span class="sxs-lookup"><span data-stu-id="22347-164">The metadata in reference assemblies continues to keep the following information:</span></span>

- <span data-ttu-id="22347-168">명시적 인터페이스 구현.</span><span class="sxs-lookup"><span data-stu-id="22347-168">Explicit interface implementations.</span></span>
- <span data-ttu-id="22347-169">명시적으로 구현된 속성 및 이벤트(해당 접근자가 가상이기 때문).</span><span class="sxs-lookup"><span data-stu-id="22347-169">Explicitly implemented properties and events, because their accessors are virtual.</span></span>
- <span data-ttu-id="22347-170">구조체의 모든 필드.</span><span class="sxs-lookup"><span data-stu-id="22347-170">All fields of structures.</span></span>

<span data-ttu-id="22347-171">참조 어셈블리에는 어셈블리 수준 [ReferenceAssembly](xref:System.Runtime.CompilerServices.ReferenceAssemblyAttribute) 특성이 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="22347-171">Reference assemblies include an assembly-level [ReferenceAssembly](xref:System.Runtime.CompilerServices.ReferenceAssemblyAttribute) attribute.</span></span> <span data-ttu-id="22347-172">이 특성을 소스에서 지정할 수 있습니다. 이렇게 하면 컴파일러가 특성을 합성할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-172">This attribute may be specified in source; then the compiler won't need to synthesize it.</span></span> <span data-ttu-id="22347-173">이 특성으로 인해 런타임은 실행용 참조 어셈블리 로드를 거부합니다. 그러나 리플렉션 전용 모드에서 로드할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="22347-173">Because of this attribute, runtimes will refuse to load reference assemblies for execution (but they can be loaded in reflection-only mode).</span></span>
