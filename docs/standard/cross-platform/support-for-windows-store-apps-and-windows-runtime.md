- <span data-ttu-id="260a1-189">.NET Framework를 사용 하 여 빌드된 Windows 런타임 구성 요소는 다른 Windows Runtimelibraries와 뚜렷한 차이점이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-189">Windows Runtime Components that are built using the .NET Framework have no apparent differences from other Windows Runtimelibraries.</span></span> <span data-ttu-id="260a1-190">예를 들어 관리 코드를 사용 하 여 네이티브 Windows 런타임 구성 요소를 다시 구현 하는 경우 두 구성 요소는 outwardly 구분할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-190">For example, if you re-implement a native Windows Runtime component by using managed code, the two components are outwardly indistinguishable.</span></span> <span data-ttu-id="260a1-191">코드 그 자체가 관리 코드이더라도 구성 요소가 관리 코드로 작성되었다는 사실은 그것을 사용하는 코드에 표시되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-191">The fact that your component is written in managed code is invisible to the code that uses it, even if that code is itself managed code.</span></span> <span data-ttu-id="260a1-192">그러나 내부적으로 구성 요소는 실제 관리 코드이며 CLR(공용 언어 런타임)에서 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-192">However, internally, your component is true managed code and runs on the common language runtime (CLR).</span></span>

- <span data-ttu-id="260a1-196">구성 요소는 Windows 8.x 스토어 앱 또는 여러 솔루션에 추가할 수 있는 재사용 가능한 구성 요소에 대 한 Visual Studio 솔루션 내의 프로젝트 일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-196">A component can be a project within a Visual Studio solution for a Windows 8.x Store app, or a reusable component that you can add to multiple solutions.</span></span>

  > <span data-ttu-id="260a1-197">구성 요소가 C# 또는 Visual Basic 에서만 사용 되는 경우에는 Windows 런타임 구성 요소로 만들 이유가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-197">If your component will be used only with C# or Visual Basic, there's no reason to make it a Windows Runtime component.</span></span> <span data-ttu-id="260a1-198">대신 일반 .NET Framework 클래스 라이브러리로 지정 하는 경우 공용 API 화면을 Windows 런타임 형식으로 제한할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-198">If you make it an ordinary .NET Framework class library instead, you don't have to restrict its public API surface to Windows Runtime types.</span></span>

- <span data-ttu-id="260a1-199">Windows 런타임 <xref:Windows.Foundation.Metadata.VersionAttribute> 특성을 사용 하 여 여러 버전에 추가 된 형식 (및 형식 내 멤버)을 식별 하 여 재사용 가능한 구성 요소의 버전을 릴리스할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-199">You can release versions of reusable components by using the Windows Runtime <xref:Windows.Foundation.Metadata.VersionAttribute> attribute to identify which types (and which members within a type) were added in different versions.</span></span>

- <span data-ttu-id="260a1-200">구성 요소의 형식은 Windows 런타임 형식에서 파생 될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-200">The types in your component can derive from Windows Runtime types.</span></span> <span data-ttu-id="260a1-201">컨트롤은 <xref:Windows.UI.Xaml.Controls.Primitives> 네임 스페이스의 기본 컨트롤 형식이 나 <xref:Windows.UI.Xaml.Controls.Button>와 같은 완성 된 컨트롤에서 파생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-201">Controls can derive from the primitive control types in the <xref:Windows.UI.Xaml.Controls.Primitives> namespace or from more finished controls such as <xref:Windows.UI.Xaml.Controls.Button>.</span></span>

  > [!IMPORTANT]
  > <span data-ttu-id="260a1-202">Windows 8 및 .NET Framework 4.5부터 관리 되는 Windows 런타임 구성 요소의 모든 public 형식이 sealed 여야 합니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-202">Starting with Windows 8 and the .NET Framework 4.5, all public types in a managed Windows Runtime component must be sealed.</span></span> <span data-ttu-id="260a1-203">다른 Windows 런타임 구성 요소의 형식은 해당 형식에서 파생 될 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-203">A type in another Windows Runtime component can't derive from them.</span></span> <span data-ttu-id="260a1-204">구성 요소에서 다형 동작을 제공하려면 인터페이스를 만들어 다형 형식에서 구현할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="260a1-204">If you want to provide polymorphic behavior in your component, you can create an interface and implement it in the polymorphic types.</span></span>
